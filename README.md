# 🪃 ブーメランチェッカー

国会議事録APIとGemini APIを使って、国会議員の過去の矛盾・ブーメラン発言を自動検出するCLIツールです。

## 仕組み

1. [国会会議録検索システムAPI](https://kokkai.ndl.go.jp/api.html)から指定した議員の発言を取得
2. Gemini API で発言間の矛盾を分析・スコアリング（原文からの一字一句抜粋も返させる）
3. **弁護人レビュー（切り抜き防止の検証層）**: 検出された各ペアについて
   - 議事録の**前後の質疑文脈**と発言者の肩書を取得し、
   - 「政治家側の弁護人」の視点で反論を全力で検討する2段目のLLMを通す
   - 誤読（他者発言の引用・仮定の話等）→ **棄却** / 状況変化等で説明がつく → **言い分付きで保持** / 弁護不成立 → **矛盾確定**
   - あわせて、LLMが返した抜粋が**議事録原文と一致するかをコードで機械検証**（捏造・改変引用の排除）
4. 結果をターミナルに表示（SNS投稿用フォーマットにも対応）。SNS/X向け出力は**公開3階層**で制御:
   - **confirmed（確定）**: 弁護人レビューでも矛盾が残った → 断定調で公開可
   - **gray（グレー）**: 説明可能と判定 → 「⚖️ 想定される言い分」を併記した両論併記フォーマットでのみ公開可
   - **非公開**: 誤読・未検証・原文一致が取れないもの

## 約束トラッカー（--promise・言 vs 行）

「発言 vs 発言」の比較は解釈の余地が大きく言葉遊びに回収されやすいため、
比較の片側を**検証可能な事実（行動・結果）**にするモード。

```bash
python main.py 野田佳彦 --promise --keyword 定数削減 --x-handle NODAYOSHI55 --sns --x
```

- **約束不履行型**（言→行）: 「必ずやる」と約束した → 実現しないまま、現在は消極・反対
- **実績否認型**（行→言）: 自分が実行・成立させた実績がある → 現在は逆方向を主張

カードは「約束/実績（原文）→ 📉事実経過 → 現在の態度（原文）→ ⚖️想定される言い分」の
3点セットで出力する。事実経過は**発言中に原文アンカーが照合できたものだけ ✅**とし、
照合できない事実は ⚠️（機械照合待ち）としてSNS出力から除外する。
弁護人レビューは「権限喪失・議会の制約・状況変化」など不履行の理由を弁護し、
本人の意思による不履行・転向は confirmed（断定調で公開可）に倒す。

## データソース（出典格付け）

| ソース | 取り込み方法 | 格付け |
|---|---|---|
| 国会会議録API | 既定（議員名＋`--keyword`） | A（完全な一次記録） |
| 本人Xポスト | `--x-handle @xxxx`（x_deep_dive 基盤・要 AUTH_TOKEN/CT0） | A（本人発信そのもの） |
| 公式サイトの全文ページ（会見録・党大会演説等） | `--source-url [YYYY-MM-DD:]URL`（複数指定可） | B（一次だが完全性は運用者の確認に依存） |

**報道記事は引用元にしない**（記事は「発言があったことを知るインデックス」に使い、
引用は必ず上記の一次ソースに錨を打つ）。出典格付けは出力に常に表示される。

## セットアップ

### 前提条件

- Python 3.10 以上
- [Gemini API キー](https://aistudio.google.com/apikey)（無料枠で動作。日次リクエスト数はツール側で自動制限）

### インストール

```bash
git clone https://github.com/your-repo/boomerang-checker.git
cd boomerang-checker
pip install -r requirements.txt
```

### 環境変数の設定

```bash
export GEMINI_API_KEY="your-api-key-here"
```

## 使い方

### 基本的な使い方

```bash
# 議員名を引数で指定
python main.py 岸田文雄

# 対話的に議員名を入力
python main.py
```

### オプション

```bash
# 取得する発言数を指定（デフォルト: 50）
python main.py 岸田文雄 --max-speeches 100

# 日付範囲を指定
python main.py 岸田文雄 --from-date 2020-01-01 --until-date 2024-12-31

# 争点キーワードで絞り込み（Xで炎上中の争点に対する立場変化を照合する使い方。
# 本文検索で絞れる分、10〜30年前の「昔の約束」まで遡って取得する）
python main.py 野田佳彦 --keyword 定数削減

# 本人Xポストと公式全文ページも照合対象に加える
python main.py 野田佳彦 --keyword 定数削減 --x-handle NODAYOSHI55 \
  --source-url "2012-11-16:https://example.gov/kaiken20121116.html"

# LLMをClaudeに切替（Claude Code CLIのサブスク枠。APIキー不要・高精度・低速）
# 公開用の本番実行は claude、安価なスキャンは gemini（既定）が目安
python main.py 野田佳彦 --promise --keyword 消費税 --llm claude

# SNS投稿用フォーマットも表示
python main.py 岸田文雄 --sns

# 弁護人レビュー（文脈検証）をスキップ（API使用量の節約用。SNS出力の信頼性は下がる）
python main.py 岸田文雄 --no-verify

# APIキーを直接指定
python main.py 岸田文雄 --api-key xxxxx
```

## プロジェクト構成

```
boomerang-checker/
├── main.py                  # CLIエントリーポイント
├── requirements.txt         # 依存パッケージ
├── README.md
└── boomerang/
    ├── __init__.py
    ├── kokkai_api.py        # 国会会議録API クライアント（発言取得＋前後文脈取得）
    ├── x_posts.py           # 本人Xポストの取り込み（x_deep_dive 基盤利用）
    ├── web_source.py        # 公式サイト全文ページの取り込み
    ├── analyzer.py          # Gemini API 矛盾分析（検出＋原文抜粋の機械検証）
    ├── verifier.py          # 弁護人レビュー（文脈検証・切り抜き防止）
    └── formatter.py         # 出力フォーマッター（公開3階層・出典格付け表示）
```

## 使用API

- **国会会議録検索システム API**: https://kokkai.ndl.go.jp/api.html
  - 国立国会図書館が提供する無料API
  - 第1回国会（1947年）からの全会議録を検索可能
- **Gemini API**: https://ai.google.dev/
  - 発言間の矛盾検出・スコアリング、および弁護人レビューに使用

## 注意事項

- 国会会議録APIは無料ですが、サーバー負荷軽減のためリクエスト間隔を設けています
- 矛盾スコアはAIによる推定値です。弁護人レビューと原文照合で誤検出を減らしていますが、公開前には必ず出典リンクから原文を確認してください
- SNS向け出力（--sns / --x）は、検証を通過した「矛盾確定」ペアのみを対象とします（切り抜き・文脈無視の指摘を受けないため）
- 本ツールは議論・分析目的であり、特定の政治的立場を支持するものではありません
